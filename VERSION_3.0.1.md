# Version 3.0.1 - 参与人筛选左右栏布局优化

**发布日期**: 2025年10月24日

## 📋 概述

本版本对参与人筛选功能进行了重大UI/UX升级，将原来的纵向分组列表改为左右栏联级布局，大幅提升了用户体验和操作效率。

## ✨ 主要功能

### 左右栏联级布局
- **左栏（部门列表）**：显示所有部门，支持点击切换
- **右栏（员工列表）**：显示当前选中部门的员工，支持多选
- **智能反馈**：实时显示部门人数和已选人数
- **视觉优化**：选中部门高亮显示，已选人数徽章提示

## 🎯 交互优化

### 部门选择
1. **默认行为**：打开时自动选中第一个部门
2. **切换部门**：点击左侧任意部门即可切换
3. **视觉反馈**：
   - 选中部门：蓝色高亮背景 `bg-[#6C63FF]`
   - 未选中部门：灰色背景 `hover:bg-gray-100`

- **已选徽章**：显示该部门已选中的人数（如"[2]"）
- **徽章样式**：
  - 选中部门：白色背景 `bg-white/20`
  - 未选中部门：蓝色背景 `bg-[#6C63FF]/10`

### 员工选择
- **多选支持**：右侧员工列表支持多选
- **选中状态**：
  - 复选框图标
  - 蓝色高亮背景
  - 动画脉冲点
- **滚动支持**：左右栏独立滚动

## 🔧 技术实现

### 核心代码

**状态管理**：
```typescript
const [selectedDepartment, setSelectedDepartment] = useState<string | null>(null);
```

**左栏部门列表**：
```tsx
<div className="w-1/3 border-r overflow-y-auto">
  {filteredGroupedOptions.map(group => (
    <button
      onClick={() => setSelectedDepartment(group.label)}
      className={selectedDepartment === group.label 
        ? 'bg-[#6C63FF] text-white' 
        : 'text-gray-700 hover:bg-gray-100'
      }
    >
      {group.label}
      {groupSelectedCount > 0 && (
        <span className="badge">{groupSelectedCount}</span>
      )}
      <div>{group.options.length}人</div>
    </button>
  ))}
</div>
```

**右栏员工列表**：
```tsx
<div className="flex-1 overflow-y-auto p-2">
  {currentGroup?.options.map(renderOption)}
</div>
```

### 性能优化
- **条件渲染**：仅渲染当前选中部门的员工
- **独立滚动**：左右栏各自滚动，互不影响
- **状态缓存**：使用 `useMemo` 缓存过滤结果

## 💡 使用指南

### 筛选参与人的步骤
1. 点击"参与人"或"按成员筛选"按钮
2. 自动选中第一个部门（如"产品管理部"）
3. 在左侧点击其他部门可切换查看
4. 在右侧勾选需要的员工
5. 可以跨部门多选员工
6. 点击"全部选择"会选中当前筛选结果的所有员工
7. 点击"取消选择"一键清空所有选择

### 搜索功能
- 在顶部搜索框输入员工姓名或邮箱
- 搜索会在所有部门中查找
- 搜索结果保持部门分组结构
- 只显示匹配的部门和员工
